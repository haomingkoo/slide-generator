# Eval Case: PDF Parser Comparison Deck

## Prompt

Create an 8-slide decision deck from the PDF parsing comparison post. Audience is an engineering lead deciding what parser pipeline to use for complaint forms. Research is disabled. Make the deck concise and use a clean surgical light theme.

## Expected Output

- Difficulty stack.
- Tool comparison matrix.
- Measured vs published benchmark separation.
- Decision tree.
- Caveats about preprocessing and corpus-specific results.

## Key Checks

- Local measurements are not mixed with published benchmarks.
- Compliance and PII constraints are visible.
- The recommendation depends on document type and constraints, not one universal winner.
