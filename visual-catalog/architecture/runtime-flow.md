# Runtime Flow

Use when the audience needs to understand what happens during execution.

## Rules

- Read left to right unless there is a strong reason not to.
- Keep the main path separate from retry/error paths.
- Use semantic colors consistently.
- Keep labels readable.
- Split into multiple slides when there are more than 10 to 12 nodes.

## Common Split

1. System context
2. Runtime happy path
3. Retry and repair points
4. Trust boundaries
5. Code snippet proof
