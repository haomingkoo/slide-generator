# Was / Is

Use when the point is a changed workflow, changed mental model, or changed product behavior.

## Structure

- Left: `Was`
- Right: `Is`
- Center or footer: implication

## Good For

- manual vs AI-assisted workflows,
- old parser vs routed parser,
- transaction scoring vs coercion scoring,
- RLHF vs DPO complexity.

## Slide Spec Fields

```json
{
  "visual_aid": "was_is",
  "was_label": "Before",
  "is_label": "Now",
  "was_points": [],
  "is_points": [],
  "implication": ""
}
```
