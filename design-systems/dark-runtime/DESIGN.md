# Dark Runtime

Use for technical runtime, architecture, security, observability, and explicit dark-theme preference.

## Palette

- Dark neutral background.
- Near-white primary text.
- Muted secondary text.
- Red, blue, green, amber, and violet only when semantic.

## Layout

- Large title and one explanatory sentence.
- One central architecture, flow, or product visual.
- Legend chips for semantic colors.
- Bottom callout cards only when they clarify the diagram.

## QA

- Diagram labels must be readable.
- Lines must not cross unnecessarily.
- Avoid red-on-dark overload.
- Split dense diagrams into multiple slides.
