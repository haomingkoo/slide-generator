# Brief

Create a six-slide source-backed deck that teaches a hackathon finalist how to structure a short demo pitch.

Audience: builders preparing for a hackathon judging session.

Use case: a three-minute live presentation followed by questions.

Goal: help the audience understand that a strong hackathon deck starts from the judging criteria, proves the hard part, and gives judges a simple path to score the project.

Constraints:

- Keep the main deck to six slides.
- Use source-backed claims only.
- Include likely judge questions and demo fallback guidance.
- Do not claim there is one universal winning formula.
- Optimize for clarity, technical credibility, and audience benefit.

Output: HTML review deck with speaker notes.
