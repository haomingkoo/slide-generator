# Content Priority

## Main Deck

- Problem and user.
- Product overview.
- Basic judging criteria.
- Demo and backup reminder.
- Clear next action.

## Backup Or Appendix

- Architecture details.
- More screenshots.
- Full submission checklist.

## Dropped From This Baseline

- Project-specific benchmark claims.
- Any guarantee that the outline wins a hackathon.

## Pruning Rule

Keep the main deck simple enough to present in a short slot.
