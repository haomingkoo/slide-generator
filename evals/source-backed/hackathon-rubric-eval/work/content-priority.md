# Content Priority

## Main Deck

- Judging reality and hook.
- Rubric-first framing.
- Main deck versus backup pruning.
- Demo path and fallback.
- Technical proof of the hard part.
- Retellable close and Q&A setup.

## Backup Or Appendix

- Full architecture diagram.
- Detailed model or API evaluation.
- Screenshots of edge cases.
- Dataset, privacy, security, or cost notes.
- Extra market or user research.

## Dropped From This Eval

- Any project-specific claim about accuracy, speed, users, revenue, or model quality.
- Any guarantee that this deck structure wins a hackathon.
- Any broad claim that all hackathons use identical judging criteria.

## Pruning Rule

If a point does not help judges score, inspect proof, remember the value, or ask a better question, move it to backup.
