# Theme Selection

Dark mode, light mode, and editorial styling are choices. They are not defaults.

## Theme Checkpoint

For polished, public-facing, client-facing, or user-reviewed decks, do not start the full render with an unnamed style. First provide a small choice set:

- 2 or 3 named theme directions,
- what each one feels like,
- when to use it,
- one preview link, screenshot, reference deck, or quick title-slide mock when available,
- the tradeoff, such as "cleaner but less expressive" or "more memorable but easier to overdo."

Ask the user to choose the closest option. If the user says "close enough" or does not want to decide, pick the safest fit for the audience and record the assumption in `work/intake-brief.md` and `work/design-contract.json`.

Keep this checkpoint lightweight. Do not build a full theme system before the user has picked a direction.

## Example Theme Menu

Use plain names the user can react to:

| Theme | Use When | Visual Cues |
|---|---|---|
| Clean Surgical Light | Consulting, executive, academic, formal proposals | Off-white background, black text, strict grid, one accent, restrained charts |
| Warm Editorial Light | Founder stories, product narratives, audience-friendly explainers | Warm paper tone, editorial type, human screenshots, generous whitespace |
| Creative Light | Pitch decks that need to feel memorable but still readable | Hard borders, poster rhythm, bold color blocks, strong hierarchy, careful footer lanes |
| Dark Runtime | Security, infrastructure, live technical demos | Dark neutral background, bright semantic labels, readable system diagrams |

When a user shares references such as Slidev themes, frontend slide repos, template screenshots, or existing decks, treat them as theme candidates. Summarize what to borrow and what to avoid before rendering.

## Clean Surgical Light

Use for:

- academic presentations,
- consulting-style recommendations,
- classroom decks,
- executive audiences,
- printed decks,
- users who prefer minimalism.

Traits:

- off-white or tinted white background,
- black or near-black text,
- one restrained accent,
- precise grid,
- thin dividers,
- charts and diagrams as the hero.

## Warm Editorial Light

Use for:

- product stories,
- portfolio posts,
- demos with human context,
- less formal audiences.

Traits:

- warm neutral background,
- editorial typography,
- generous whitespace,
- selective product screenshots or mockups.

## Dark Runtime

Use for:

- system architecture,
- security,
- observability,
- technical runtime flows,
- dramatic demo moments,
- explicit user preference.

Traits:

- dark neutral background,
- high contrast labels,
- semantic accent colors,
- readable diagrams,
- no tiny labels.

## Brand-Derived

Use when brand assets exist. Extract:

- primary and secondary colors,
- typography,
- logo rules,
- density,
- tone,
- visual references.

Do not override brand with a generic trendy deck style.
